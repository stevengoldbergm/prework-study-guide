HTML

<head>

  • The head element contains information about the webpage itself.
    • This includes metadata and associated tags, as well as details important for Search-Engine Optimization (SEO).
    • The head element is not visible to the end-user.
    • When using an external CSS style sheet, the link must be placed in the <head> element.

<body>

  • The body element represents the visible content shown to the user, including such items as:
    • Headers
    • Paragraphs of text
    • Images, and
    • External links

<, >, and &

  • There are some text items that require special characters to write in HTML code:
    • The "less than" character (<) is written as '&lt'
    • The "greater than" character (>) is written as '&gt'
    • The ampersand character (&) is written as '&amp'
  • A full list of similar commands can be found here

CSS

Margins and Padding

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Number values for both margin and padding address the top/bottom of the element first, followed by the left/right of the element.
    • E.g. Adding the code "padding: 0 5px" to your style sheet will result in the left and right sides of the element have a 5 pixel internal buffer, while the top and bottom edges have no buffer at all.

Custom Class Elements

  • A custom class element can be added to the style sheet by using the .[class] element.
  • The cards on this webpage use a custom element called ".card" for example.
  • By applying the custom class to a section of the HTML and identifying it as a "css-section," HTML kows to use the formatting data in the custom class for the whole section.

Git

Basic Git Commands

  • git -h: Brings up the help screen for git
  • git status: Checks the branch you are currently working in and identifies any untracked files.
  • git branch: Shows all available branches on your local machine.
  • git branch branchName: Creates a new local branch without changing your current branch
  • git branch -d branchName: Deletes an existing branch.
  • git checkout branchName: Allows you to checkout the specific existing branch
  • git checkout -b 'branchName': Creates a new branch off the current branch and checks it out.

Pushing changes to Github

  • git add -A: Stages all files to commit.
  • git commit -m "commit message here": Commits changes to GitHub with comments.
  • git pull origin main: Syncs main branch with GitHub to ensure files are updated before pushing changes.
  • git push origin branchName: pushes updated commits to the specified branch.

NOTE: Additional considerations may need to be made regarding the above code.


Pulling a Branch from GitHub that isn't on the Local Machine

  • git checkout main: Ensures that you are working on the main branch to start
  • git status: Checks the status of your code to make sure the master (or main) branch is up to date
  • git pull origin main: Pulls the main branch to integrate any updates (if necessary).
  • git fetch: Updates the list of branches on the working machine to match the branches in the remote repository.
  • git branch -a: Shows all local and remote branches
    • This code allows us to see all local and remote branches.
    • git branch -r: Shows only the remote branches.
    • git branch: Shows only the available local branches.
  • git checkout nameOfRemoteBranch: Include all text after "origin" in the branch results to connect to the remote branch.
  • git pull: Updates your current branch files to match those from the remote repository.

JavaScript

Basic Notes

  • JavaScript is used for all of the interactive parts of a website.
  • When connecting a .js file to your webpage, the script link must be placed at the end of the body code, similar to how .css files must be linked at the end of the head code.
  • A variable is a named container that allows us to store data in our code. It can be a static numeric value, a string of text characters, or even an array of different values!
  • Control flow is the order in which a computer executes code in a script - in JavaScript, the order goes from top to bottom!

Variables, Objects, Operators, and Comments

  • Variables:
    • Variables are containers that store values.
    • Variables must be set by indicating a name and a value
    • There are three different ways to set variables:
      • var: var myFirstVariable = 'firstName ';
      • const: const mySecondVariable = 'lastName';
      • let: let myCombinedVariables = x + y;
    • When should I use each variable?
      • var: For older browsers. JavaScript did not use let or const until 2015.
      • const: For static variables in newer browsers. As a rule of thumb, const should work for most variables
      • let: For variables where the value is expected to change. Let works well for relational expressions using existing declared variables (e.g. let z = x + y)
  • Objects:
    • Everything in JS is an object and can be stored in a variable
  • Operators
    • An operator is a mathematical symbol that produces a result based on two values.
    • You can find more information on operators here
  • Comments
    • // Two forward slashes will indicate that an entire line is a comment.
    • /* Anything between the two asterisks, including multiple lines of code, will be indicated as a comment */

Dynamic Variables

  • When defining a variable, JavaScript will determine the type dynamically based on the value you set.
    • var foo = 42; //foo is now a number
    • const foo = 'bar' //foo is now a string
    • let foo = true //foo is now a boolean

    Note: Once a variable is declared with let, it cannot be redeclared.


Functions

  • Functions are a way of packaging functionality that you wish to reuse.
  • Functions allow you to perform specific coded actions when called, allowing you to create dynamic effects with smaller amounts of code.

If Statements

  • Similar to other coding languages, the If statement in JavaScript checks for a specific condition, and executes code based on whether or not the results match another indicated value.
  • This is known as conditional logic.
  • Example Syntax:
    • If (x + 1 == 4) {This code will execute}
    • Else If (x + 1 == 5) {This second code will execute}
    • Else {This alt code will execute}

Arrays

  • Arrays are variables that store multiple values.
  • An array can be thought of as being like a spreadsheet, where each box on the spreadsheet indicates an index number value
  • Index values in arrays start at 0, and then move up by 1 for each value present in the array.
  • Example Syntax:
    • var shapes = ['triangle', 'square', 'pentagon', 'hexagon']
    • The above variable shapes is an array. The strings, from left to right, indicate index values from 0 to 3.
    • To use this variable, it needs to be called with an index value. (e.g. shapes[0])

Note: You can use a previously defined function as an array's value, so long as the function's result is an index number. The random subject selector in the console uses this kind of logic to generate a new result on a page refresh!


For Loops

  • A For loop uses predictable patterns of indices to perform a task on all items in an array: this is also known as recursion
  • For loops all contain the following three parts:
    • A starting point.
    • A break condition.
    • An iterative value
  • Example Syntax:
    • for (var x = 0; x < 4; x++) {This code will execute.}
    • The above code indicates that starting at index 0, so long as the x variable is less than 4, the code will be executed and the x value will be increased by 1.
